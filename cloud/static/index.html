Cognitive States

Four states. One goal — catch overload early.

ZenNode scores your brain every 5 seconds and classifies it into one of four states.

🟢
Flow
Score 0 – 30
Deep focus, everything clicking. ZenNode stays completely invisible — no interruptions, no nudges.
🟡
Friction
Score 31 – 60
Frequent edits, rising undo rate. Working hard but managing. A gentle heads-up may appear.
🟠
Fatigue
Score 61 – 80
Long idle gaps, slowed typing. Brain is tiring. Actionable nudges begin — focus tips, tab cleanup.
🔴
Overload
Score 81 – 100
Rapid tab-cycling, pasting AI blocks without review. Full intervention: amber theme + LLM message + breathing prompt.

How It Works

From keystroke to intervention in 5 seconds.

ZenNode runs a tight loop — collect, score, act — every 5 seconds, entirely in the background.

1
Collect behavioral signals
The TraceCollector counts keystrokes, backspaces, tab switches, undo actions, idle time, and paste events every 5 seconds. It records counts only — never the content of what you type.
2
Score locally with EMA
The scorer normalizes each signal into a 0–100 metric, applies weights, and computes a cognitive load score. An Exponential Moving Average (α=0.6) smooths the score to prevent jitter. All math runs in TypeScript — no server needed.
3
Classify the state
The score is mapped to one of four states: Flow (0–30), Friction (31–60), Fatigue (61–80), or Overload (81–100). The Zen Bar updates instantly in the status bar.
4
Intervene at the right moment
Depending on the state, ZenNode triggers the appropriate response: theme shift at overload, notification nudge at fatigue, LLM-generated message via the optional cloud at overload.
5
Sync to team (optional)
Every ~50 seconds, an anonymized session summary is pushed to the cloud. Managers see aggregated team health — never individual raw data. You control the connection; it's opt-in only.
Scoring Algorithm

Transparent math. No black box.

The cognitive load score is a weighted sum of 5 normalized behavioral metrics, smoothed with EMA.

S = 0.25·SwitchRate + 0.20·ErrorRate + 0.25·UndoRate + 0.15·IdleRatio + 0.15·PasteRatio
SwitchRate
25%
Tab switches per minute. Strongest signal of context-switching overload.
ErrorRate
20%
Backspaces ÷ total keystrokes. Indicates struggle and friction.
UndoRate
25%
Undos per minute. Directly correlates with decision fatigue.
IdleRatio
15%
Idle seconds ÷ total seconds. Signals disengagement or exhaustion.
PasteRatio
15%
Pasted chars ÷ total chars. Flags AI dump without review behavior.

Privacy

Privacy is the foundation, not a feature.

Mental health tooling only works if developers trust it. Every privacy claim below is enforced in code — not just policy.

🖥️
Scoring runs 100% locally
The TypeScript scorer runs inside the extension process. No data leaves your machine for cognitive scoring. No server required.
🔢
Counts only, never content
ZenNode records how many keystrokes, never what you typed. File names, variable names, and source code are never accessed.
🤖
LLM sees metadata, not code
The AI receives only anonymized behavioral numbers — e.g., "tabSwitches: 18, undos: 12". Your source code is never sent anywhere.
👤
Team sync is opt-in and anonymized
Cloud sync only activates when you choose to connect. Sessions are identified by a random anonymous_id — never your name or email.
📵
No telemetry, ever
ZenNode has no analytics, no crash reporting, no phone-home. It cannot phone home because it has no such code.
🔍
Fully auditable
The entire codebase is open source. Every privacy claim above can be verified line-by-line on GitHub.
Setup

Up and running in 2 minutes.

Requirements: VS Code ≥ 1.85 · Node.js ≥ 18 · Python ≥ 3.10 (cloud only)

1
Clone & install
Clone the repo and install Node.js dependencies.
2
Build the extension
Compile TypeScript and package as a .vsix file.
3
Install in VS Code
Install the .vsix and reload VS Code. The Zen Bar appears.
4
Start the cloud (optional)
Run the FastAPI backend for LLM interventions and team sync.
# 1. Clone and install
git clone https://github.com/ryadavtmc/zen-node.git
cd zen-node
npm install

# 2. Build and install the extension
npm run compile
vsce package
code --install-extension zennode-0.1.0.vsix

# 3. Start the cloud backend (optional)
cd cloud
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --host 127.0.0.1 --port 8421 --reload
